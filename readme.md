# Car Parking Occupancy Detection

A real-time parking lot occupancy detection system using classical computer vision and machine learning. The system processes a parking lot video feed, identifies individual parking spots via a binary mask, and classifies each spot as **empty** or **occupied** using a trained Support Vector Machine (SVM) — overlaying live status on the video with color-coded bounding boxes.

|---|---|
| Green rectangle | Red rectangle |

```

```

## How It Works

The system is built around three core ideas:

### 1. Mask-Based Spot Detection
A binary mask image (white blobs on a black background) defines the location of each parking spot. OpenCV's `connectedComponentsWithStats` extracts the bounding box `(x, y, w, h)` for every white blob — each blob corresponds to one parking spot.

### 2. Change-Aware Frame Processing
To avoid redundant inference on every frame, the system only re-classifies a parking spot if its pixel-mean intensity changes significantly compared to the previous sampled frame. Spots where:

```
diff / max_diff > 0.4
```

are flagged as "potentially changed" and re-evaluated. This threshold-based filter makes the system efficient even on long videos.

### 3. SVM Classifier
Each parking spot crop is resized to `15×15×3` and flattened into a 675-dimensional feature vector. A pre-trained SVM (trained with GridSearchCV over RBF kernel parameters) predicts whether the spot is **empty (0)** or **not empty (1)**.

## Training the Model

If you want to train the SVM from scratch on your own data:

### 1. Prepare Training Data

Organize cropped parking spot images into two folders:

```
data_for_model/
├── empty/         # Images of empty parking spots
└── not_empty/     # Images of occupied parking spots
```

### 2. Update the Input Path in `model.py`

```python
input_dir = "path/to/data_for_model"
```

### 3. Run the Training Script

```bash
python model.py
```

This will:
- Load and resize all images to `15×15`
- Perform an 80/20 train-test split (stratified)
- Run `GridSearchCV` over SVM hyperparameters:
  - `C`: [1, 10, 100, 1000]
  - `gamma`: [0.01, 0.001, 0.0001]
- Print classification accuracy on the test set
- Save the best model to `model.p`

**Expected output:**
```
96.43% of samples were correctly classified.
```

## Technical Details

### Spot Detection Pipeline

```
mask_1920_1080.png
        │
        ▼
cv2.connectedComponentsWithStats()
        │
        ▼
List of bounding boxes [(x1, y1, w, h), ...]
        │
        ▼
Crop each spot from the video frame
        │
        ▼
Resize to 15×15×3 → Flatten to 675-d vector
        │
        ▼
SVM.predict() → EMPTY (0) or NOT_EMPTY (1)
        │
        ▼
Draw green/red rectangle on frame
```

### Frame Sampling Strategy

- Inference is not run on every frame — only every `step = 30` frames.
- On each sampled frame, the pixel-mean difference between the current and previous frame is computed per spot.
- Only spots with `diff / max_diff > 0.4` are re-classified, saving computation.

### SVM Details

| Parameter | Value |
|-----------|-------|
| Kernel | RBF (default SVC) |
| Input size | 15 × 15 × 3 → 675 features |
| Tuning method | GridSearchCV |
| C values | [1, 10, 100, 1000] |
| Gamma values | [0.01, 0.001, 0.0001] |
| Test split | 20% (stratified) |
